# 整改任务清单

> 基于 `docs/project-issues-report.md` 的问题分析，整理出的可执行整改项。建议按优先级分阶段推进。

## 目标

- 降低 `GraphPage` 和 `useGraph` 的复杂度
- 收敛 Tab / 全局模式的状态来源
- 将导航、展示、持久化职责拆开
- 提升可测试性和长期可维护性

---

## P0 - 必须先做

### 1. 拆分 `GraphPage` 的职责

**目标**
- 将页面布局、导航同步、右侧面板拖拽、搜索同步等逻辑拆分出去

**任务内容**
- 抽离页面级布局容器
- 将右侧面板宽度拖拽逻辑拆成独立 hook
- 将日志埋点和状态恢复逻辑拆成独立 hook
- 保持 `GraphPage` 只负责组合子组件

**验收标准**
- `GraphPage` 中明显减少副作用和条件分支
- 页面文件更接近“组合层”而不是“控制层”

---

### 2. 收敛 Tab / 全局模式的数据读取方式

**目标**
- 避免在多个位置手工判断 `tabId` 后分别读写不同 store

**任务内容**
- 统一封装“当前上下文状态”的读取入口
- 明确哪些字段由 `tabStore` 负责，哪些字段由 `appStore` 负责
- 减少 `GraphPage`、`useGraph`、面包屑、详情面板中的兼容分支

**验收标准**
- 同类状态只存在单一主来源
- 不再需要到处手写 `tabId ? xxx : yyy` 的分支

---

### 3. 把导航行为从 UI 层抽离

**目标**
- 让面包屑、按钮、菜单等 UI 组件只负责触发，不负责组织导航规则

**任务内容**
- 提炼统一的导航服务或导航 hook
- 将路径计算、历史跳转、回到根级等逻辑集中处理
- UI 组件仅接收已计算好的展示数据和回调

**验收标准**
- UI 组件不再直接拼装复杂导航逻辑
- 导航规则改动时，主要修改集中在单一模块中

---

## P1 - 尽快推进

### 4. 拆分 `useGraph` 的职责

**目标**
- 将当前超大的 hook 拆成多个职责更单一的 hook 或服务层

**任务内容**
- 把加载/保存逻辑与 React Flow 事件处理分离
- 把搜索高亮单独拆出
- 把节点/边 CRUD 逻辑和导航逻辑分离
- 将 dirty 状态管理独立出来

**验收标准**
- `useGraph` 不再承担所有图谱行为
- 每个 hook 的职责都能被清楚描述

---

### 5. 减少 store 之间的双向同步桥接

**目标**
- 降低 app store、tab store、room store 之间相互补写造成的不一致风险

**任务内容**
- 明确同步方向，尽量只保留单向同步
- 删除不必要的“兼容写入”逻辑
- 对必须保留的兼容层添加清晰注释和收敛计划

**验收标准**
- 同一个状态字段不会在多个 store 中反复写入
- 不需要依赖多个 effect 才能保持一致

---

### 6. 梳理副作用和时序控制

**目标**
- 减少 `queueMicrotask`、ref 兜底、手动监听带来的理解成本

**任务内容**
- 统一梳理哪些副作用必须延后执行
- 将临时规避写法替换为更稳定的状态流
- 对关键 effect 添加明确的设计说明

**验收标准**
- 副作用链路更清晰
- 读代码时能看懂“为什么要这样安排时序”

---

## P2 - 中期优化

### 7. 统一领域模型和类型接口

**目标**
- 让 TypeScript 真正承载业务语义，而不是只做字段检查

**任务内容**
- 定义统一的路径、房间、选中态、边样式等领域类型
- 避免组件直接依赖 store 的原始内部结构
- 抽出可复用的类型和转换函数

**验收标准**
- 核心状态有统一类型定义
- 业务概念不会在不同模块里出现多个表达方式

---

### 8. 补齐关键路径测试

**目标**
- 提升导航和状态同步的回归保护能力

**任务内容**
- 为路径推导和历史跳转补单测
- 为 Tab / 全局模式切换补测试
- 为关键加载/保存流程补测试

**验收标准**
- 核心导航行为可通过自动化测试验证
- 关键回归点有明确保护

---

### 9. 优化兼容迁移策略

**目标**
- 避免“临时兼容层”长期化，变成新的主逻辑

**任务内容**
- 给兼容逻辑标记明确的移除条件
- 记录兼容层保留原因
- 每轮迭代评估是否可以收敛

**验收标准**
- 兼容层有明确生命周期
- 不会无限叠加新的兼容分支

---

## 建议执行顺序

1. 先拆 `GraphPage` 和导航入口
2. 再收敛 Tab / 全局模式的数据来源
3. 同步拆分 `useGraph`
4. 然后整理类型和测试
5. 最后清理兼容层
