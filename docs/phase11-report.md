## 2. 修复了哪些 Bug

本次维护共修复 **5 个 Bug**：

### Bug 1: Vue 生命周期 hook 错误使用
- **文件**: `src/composables/useNodeBadges.js`
- **问题**: 使用 `onUnmounted` 但 composable 是从 `onMounted` 调用，不在 setup 阶段
- **修复**: 改用 `onScopeDispose` (Phase 10 确立的正确模式)
- **代码变更**:
  ```javascript
  // 修改前
  import { onUnmounted } from 'vue'
  onUnmounted(() => { ... })
  
  // 修改后
  import { onScopeDispose } from 'vue'
  onScopeDispose(() => { ... })
  ```

### Bug 2: Pinia store 数组变异
- **文件**: `src/stores/room.js`
- **问题**: `closeTab` 中使用 `this.tabs.splice(idx, 1)` 变异数组
- **修复**: 改用 immutable 模式 `this.tabs = this.tabs.filter(...)`
- **代码变更**:
  ```javascript
  // 修改前
  this.tabs.splice(idx, 1)
  
  // 修改后
  this.tabs = this.tabs.filter((_, i) => i !== idx)
  ```

### Bug 3: 核心服务的监听器移除变异
- **文件**: `src/core/git-backend.js`
- **问题**: `onStatusChange` 的 cleanup 函数使用 `splice` 移除监听器
- **修复**: 改用 `filter` 创建新数组
- **代码变更**:
  ```javascript
  // 修改前
  const idx = _listeners.indexOf(fn)
  if (idx !== -1) _listeners.splice(idx, 1)
  
  // 修改后
  _listeners = _listeners.filter(cb => cb !== fn)
  ```

### Bug 4: Vue 组件数组直接长度赋值为 0
- **文件**: `src/components/StylePanel.vue`
- **问题**: `clearRecentColors` 使用 `.length = 0` 清空数组
- **修复**: 改用 `.splice(0, length)` 方法调用
- **代码变更**:
  ```javascript
  // 修改前
  recentFontColors.length = 0
  
  // 修改后
  recentFontColors.splice(0, recentFontColors.length)
  ```

### Bug 5: Vue 组件数组 splice 变异
- **文件**: `src/components/StylePanel.vue`
- **问题**: `toggleFontStyle` 使用 `splice` 移除数组元素
- **修复**: 改用 `filter` 创建新数组
- **代码变更**:
  ```javascript
  // 修改前
  if (idx >= 0) parts.splice(idx, 1)
  else parts.push(style)
  styles.fontStyle = parts.join(' ')
  
  // 修改后
  const newParts = idx >= 0 ? parts.filter((_, i) => i !== idx) : [...parts, style]
  styles.fontStyle = newParts.join(' ')
  ```

## 6. 注意事项

### 编码规范提醒
- **Immutability 原则**: 永远创建新对象，永远不要变异现有对象
- **Vue 生命周期**: composable 中的清理逻辑应使用 `onScopeDispose` 而非 `onUnmounted`（后者仅在 setup 阶段生效）
- **Pinia actions**: 必须使用 immutable 模式 (`filter`/`map` 返回新数组)，禁止使用 `splice`/`pop`/`push` 等变异方法

### 后续建议
- 建议在全项目范围扫描其他可能存在 `splice` 变异的 Pinia actions
- 建议检查其他 composables 是否正确使用 `onScopeDispose`
- 建议添加 ESLint rule 强制 immutability 检查
