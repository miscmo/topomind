## 【四、功能优化详情】

### 4.3 Bug Fix: E2E 测试 `page.evaluate()` 挂起

**问题描述**：`room-nav.electron.spec.ts` 中双击节点后调用 `page.evaluate()` 挂起 30 秒超时。React Flow 的 `onNodeDoubleClick` 回调可能在 React 合成事件系统内部触发状态更新，`page.evaluate()` 与 React 的批量更新机制产生死锁。

**原因分析**：`page.mouse.dblclick()` 触发 Playwright 的 DOM 事件派发，React Flow 内部同步调用 `onNodeDoubleClick`，事件处理函数中可能存在 Promise 未 resolved 的情况，导致 `page.evaluate()` 在同一事件循环中等待。

**解决方案**：
1. 改用 Playwright 的 `Locator.dblclick({ force: true })` 替代原生 `page.mouse.dblclick()`，绕过 Chromium 输入事件层
2. 移除重复的 `const node` 声明避免变量遮蔽

**验证方式**：运行 `npx playwright test e2e-electron/room-nav.electron.spec.ts --grep "钻入子房间后显示知识库根与当前房间"` 验证测试通过。
