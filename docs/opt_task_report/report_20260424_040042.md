## 四、功能优化详情

### 1. Zustand Store 完整测试覆盖

**问题描述**: `appStore` 和 `roomStore` 承担核心状态管理职责，但此前无单元测试，边界行为（如 `goBack` 在不同历史长度下的行为）缺乏自动化验证。

**解决方案**: 为两个 store 各编写完整的测试套件，共 40 个测试用例：
- `appStore`（16 测试）：视图切换、节点选中、边模式、Git 面板、右键菜单、网格、搜索
- `roomStore`（24 测试）：进入/退出房间、历史导航（`goBack`/`navigateToHistoryIndex`/`goToRoom`）、全局视图判断、面包屑生成

**验证方式**:
```bash
$ npx vitest run src/test/storeApp.test.ts
✓ 40 passed
```

### 2. goBack 边界行为澄清

**问题描述**: `roomStore.goBack()` 从长度为 N 的 history 栈中弹出最后一项，设置 current 为 `newHistory[N-2]`（或全局）。团队对此行为存在误读。

**测试澄清的行为**:

| 历史长度 | 弹出项 | current 设置 |
|---------|-------|------------|
| 0 | null | 不变（返回 null） |
| 1 | history[0] | null（全局），history 变空 |
| ≥2 | history[last] | newHistory[last-1]（倒数第二条） |

### 3. useLayout ELK 布局逻辑测试

**问题描述**: `computeLayoutImpl` 负责将节点数据转换为 ELK 图并解析布局结果，未覆盖边界场景。

**覆盖场景**:
- 空输入、ELK 异常、ELK 结果无 children
- DOWN/RIGHT 方向、跳过 undefined x/y 节点
- 节点尺寸默认值与measured fallback
- parent-based edges 传递、layout direction 传递
- 大规模节点（200+）触发 MIN_SPACING 优化
