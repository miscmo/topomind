#### HIGH — HomePage.tsx 内存泄漏（定时器堆积）

**问题描述**：`enterLogTimers` 使用 `useRef(new Map())` 持有定时器映射。当知识库卡片删除后，对应的定时器条目从未从 Map 中移除，Map 只增长不收缩，造成组件生命周期内的内存泄漏。

**修复**：在 `HomePage.tsx` 中添加 `useEffect` 卸载清理，在组件卸载时遍历并清除所有待决定时器：

```typescript
useEffect(() => {
  return () => {
    for (const timer of enterLogTimers.current.values()) {
      clearTimeout(timer)
    }
    enterLogTimers.current.clear()
  }
}, [])
```

**修复文件**：`src/components/HomePage.tsx` 行 61 之后

---

#### HIGH — StyleSection.tsx 冗余写入（重复 useEffect）

**问题描述**：存在两个独立的写入路径：

1. `updateDefaultStyle()` 手动调用 `storage.writeConfig({ defaultEdgeStyle: next })`
2. 一个独立的 `useEffect(() => { storage.writeConfig({ defaultEdgeStyle }) }, [storage, defaultEdgeStyle])`

两者都写入相同数据，第二个 `useEffect` 在快速切换样式时会触发大量即时磁盘写入，并可能覆盖 `updateDefaultStyle` 中已写入的值（竞态）。

**修复**：删除独立的 `useEffect`，仅保留 `updateDefaultStyle` 中的写入路径。

**修复文件**：`src/components/DetailPanel/StyleSection.tsx` 行 46-48

#### MEDIUM — DetailPanel.tsx handleSave 无异常处理

**修复**：添加 `try/catch`，错误时通过 `logger.catch` 记录，并导入 `logger`。

**修复文件**：`src/components/DetailPanel/DetailPanel.tsx` 行 100-109
