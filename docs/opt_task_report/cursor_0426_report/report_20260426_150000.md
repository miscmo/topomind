## 一、发现的问题清单

### 1. 功能问题
1. `GitPanel` 功能完整，能展示状态、提交、推送、拉取、获取远端信息，但状态解析字段较宽，依赖上层 `useGit` 返回结构稳定。
2. `Toolbar` 功能完整，但缺少重置缩放或适配视图的快捷入口，用户只能缩放而不能快速回到默认视图。
3. `ContextMenu` 功能完整，支持节点/边/空白区域右键，但 pane 模式仅提供“新建子节点”，语义上较窄。

### 2. 逻辑 Bug
1. `GitPanel` 在读取 status 时将 `modified/untracked/remote` 等字段通过强制断言解释，若返回结构变化，badge 状态可能显示不准确。
2. `GitPanel` 的 `diffStat` 中 `insertions` / `deletions` 恒为 0，当前不影响功能，但属于逻辑未完成的统计占位。
3. `ContextMenu` 的 pane 模式直接把 `onNewChild('')` 传给上层，依赖上层处理空 target，逻辑可用但约定较隐式。
4. `Toolbar` 仅提供 zoom in/out，缺少 reset / fit view 入口，用户如果缩放到很远，恢复路径不够直接。

### 3. 安全风险
1. `GitPanel` 展示 remote URL 时做了简单字符串替换，没有额外脱敏逻辑；虽然是常见显示方式，但若未来带 token 的 URL 被写入，可能泄露敏感信息。
2. `ContextMenu` 触发的新建/重命名/删除操作都由上层回调处理，若上层权限校验缺失，会间接放大误操作风险。
3. `Toolbar` 通过 `useReactFlow` 直接控制视图，若后续扩展更多视图控制能力，需要继续确保与页面状态一致。

### 4. 内存泄漏
1. `ContextMenu` 会根据 visible 状态动态注册 document/window 监听器，当前清理完整，但属于高频组件，后续新增事件需继续检查解绑。
2. `GitPanel` 在面板打开时触发多项异步加载，虽无明显定时器，但未来若增加轮询，需要注意清理。
3. `Toolbar` 本身无明显泄漏点，但依赖 React Flow 上下文，若挂载链变化要注意销毁顺序。

### 5. 体验问题
1. `GitPanel` 的状态 badge 和变更 badge 组合虽然可用，但 diffStat 统计信息不够丰富，用户对变化规模的判断有限。
2. `Toolbar` 没有“回到中心/复位视图”按钮，缩放后恢复不够方便。
3. `ContextMenu` 在 pane 里只有“新建子节点”，无法从空白区直接发起更多常用操作，功能入口偏少。
4. `GitPanel` 的远端 URL 展示做了简化显示，但未提供复制或跳转入口，体验上还有提升空间。

### 6. 工程化问题
1. `GitPanel` 中状态解析的类型转换较多，后续如果 `useGit` 返回结构变化，需要同步调整这里的强制断言。
2. `ContextMenu` 的菜单条目通过内联数组组装，功能完整，但后续菜单项增多时可维护性会下降。
3. `Toolbar` 目前功能较单一，未来扩展时建议拆出更多操作而不是继续堆在一个匿名按钮组里。

### 7. TS 类型问题
1. `GitPanel` 里存在较多 `as Record<string, unknown>`、`as DiffFile[]` 之类的断言。
2. `ContextMenu` 的 `MenuEntry` 联合类型可用，但菜单 action 的返回值兼容 `Promise<void>`，调用链需要持续注意。
3. `Toolbar` 的 props 很少，类型简单，但功能扩展时应保持明确接口。

## 四、功能优化详情

### 问题 1：Toolbar 缺少复位视图入口
- 问题描述：只有缩放，没有回到默认视图的快捷操作。
- 原因分析：当前只实现了 zoom in/out。
- 解决方案：后续可考虑补一个 fit/reset 入口，但需和 React Flow 现有视图逻辑对齐。
- 验证方式：缩放后是否能一键恢复默认可视范围。

### 问题 2：GitPanel 的统计信息不完整
- 问题描述：insertions/deletions 固定为 0。
- 原因分析：当前只关注文件状态统计，未接入更细 diff 统计。
- 解决方案：后续若要展示更完整统计，可补 diff 数据解析。
- 验证方式：展示的 badge 是否与仓库实际状态一致。

### 问题 3：ContextMenu pane 模式动作较少
- 问题描述：空白区右键仅支持新建子节点。
- 原因分析：当前产品设计如此，但入口偏少。
- 解决方案：若业务需要，可考虑增加更常用的空白区操作。
- 验证方式：右键空白区是否能正常创建节点。
