### 本次新增修复

#### Bug 5: `edgesMapRef` 数据不一致

**严重级别**: HIGH
**文件**: `src/hooks/useGraph/graphOperations.ts`
**描述**: `edgesMapRef` 在 `updateEdgeRelation` 中未被更新，导致后续通过 `edgesMapRef` 读取边数据的操作返回过期数据。

**修复**: 在 `updateEdgeRelation` 的 `setState` 回调中添加 `edgesMapRef.current = new Map(edges.map((e) => [e.id, e]))`。

```typescript
// 修复前
setState((prev) => {
  const edges = prev.edges.map(...)
  edgesRef.current = edges
  rebuildMaps(prev.nodes, edges)  // rebuildMaps 更新 nodesMapRef 但不更新 edgesMapRef
  return { ...prev, edges }
})

// 修复后
setState((prev) => {
  const edges = prev.edges.map(...)
  edgesRef.current = edges
  edgesMapRef.current = new Map(edges.map((e) => [e.id, e]))
  return { ...prev, edges }
})
```

#### Bug 6: `applyNodeRemoveChanges` 未清理已删除节点的连线

**严重级别**: HIGH
**文件**: `src/hooks/useGraph/graphOperations.ts`
**描述**: React Flow 的 `onNodesChange` 中 `type: 'remove'` 的变更只过滤了节点，未同步删除连接到这些节点的边，导致 `orphaned edges`（悬空边）残留在状态中。

**修复**: 在 `applyNodeRemoveChanges` 中添加边过滤逻辑：

```typescript
const applyNodeRemoveChanges = (changeIds: string[]) => {
  const removedSet = new Set(changeIds)
  setState((prev) => {
    const nodes = prev.nodes.filter((n) => !changeIds.includes(n.id))
    // Remove edges connected to deleted nodes to avoid orphaned edges
    const edges = prev.edges.filter((e) => !removedSet.has(e.source) && !removedSet.has(e.target))
    nodesMapRef.current = new Map(nodes.map((n) => [n.id, n]))
    edgesMapRef.current = new Map(edges.map((e) => [e.id, e]))
    nodesRef.current = nodes
    edgesRef.current = edges
    return { ...prev, nodes, edges }
  })
  const dirPath = getActiveNavState().roomPath
  if (dirPath) scheduleSave(dirPath)
}
```
