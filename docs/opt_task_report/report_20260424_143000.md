## 四、功能优化详情

### Issue #1: Blob URL 内存泄漏（App 退出时从不释放）

**问题描述**
`ImageUrlRegistry` 类（`src/core/storage.ts`）实现了 `revokeAll()` 方法，可在 App 退出时批量释放所有 Blob URL，防止内存泄漏。但该方法从未被任何代码调用。

**原因分析**
`Store.revokeAllImageUrls()` 已在 `storage.ts` 中导出，但 App.tsx 没有任何 unmount cleanup 逻辑。`ImageUrlRegistry` 实例 `_imageUrlRegistry` 持有私有引用，外部无法直接访问，只能通过 `Store.revokeAllImageUrls()` 调用 `revokeAll()`。

**解决方案**
在 App.tsx 中添加 unmount useEffect，直接调用 `Store.revokeAllImageUrls()`：
```tsx
useEffect(() => {
  return () => {
    Store.revokeAllImageUrls()
  }
}, [])
```

**验证方式**
`tsc --noEmit` 通过；`vite build` 成功；手动 Electron 环境中打开含图片的知识库后关闭，Blob URL 无残留引用。

---

### Issue #2: useGit 清理定时器竞态条件

**问题描述**
`useGit()` 使用模块级 `let cleanupRegistered = false` 布尔标志管理 Git 缓存清理定时器。当多个组件同时调用 `useGit()` 时，第一个组件卸载调用 `stopGitCacheCleanup()`，将 timer 停止并置 null，后续仍在使用的组件失去清理定时器，导致缓存永不清理（资源浪费）或 timer 引用指向已清理对象（潜在内存问题）。

**原因分析**
布尔标志只记录"是否已注册"，无法表达"被多少调用者共享"。当有 N 个 `useGit()` 调用者时，第 N-1 个卸载会停止 timer，而前 N-1 个实际上仍需要它。

**解决方案**
将布尔标志替换为引用计数器 `let cleanupRefCount = 0`：
```tsx
let cleanupRefCount = 0

useEffect(() => {
  cleanupRefCount++
  if (cleanupRefCount === 1) {
    startGitCacheCleanup()
  }
  return () => {
    cleanupRefCount--
    if (cleanupRefCount === 0) {
      stopGitCacheCleanup()
    }
  }
}, [])
```

**验证方式**
`tsc --noEmit` 通过；`npx vitest run` 107 测试全通过；多个 `useGit()` 并存的组件卸载顺序不影响 timer 生命周期。

### Issue #3: DetailPanel handleSave 过期闭包

**问题描述**
`handleSave` 函数使用 `selectedNode?.data.path` 获取节点路径，但 `selectedNode` 是渲染时捕获的 prop。当用户在 React Flow 画布上切换选中节点（不通过 DetailPanel 内部流程），DetailPanel 可能不会触发重渲染，导致 `selectedNode` 数据陈旧。保存操作会写入错误的路径，破坏文件数据。

**原因分析**
React Flow 的节点选择事件通过 React context 传递，不一定触发 DetailPanel 重渲染（如果 DetailPanel 未订阅对应状态）。`selectedNode` 在 DetailPanel 渲染时从 props 解构获得，后续 `handleSave` 闭包中引用的是渲染时的旧值。

**解决方案**
`handleDelete` 函数已正确使用 `graph.nodesMapRef.current.get(selectedNodeId)` 获取最新数据。对 `handleSave` 应用相同模式：
```tsx
const handleSave = () => {
  if (!selectedNodeId) return
  const node = graph.nodesMapRef.current.get(selectedNodeId)
  const path = node?.data.path ?? selectedNodeId
  const label = node?.data.label
  storage.writeMarkdown(path, markdown)
  logAction('内容:保存', 'DetailPanel', { nodePath: path, label })
  setEditMode(false)
}
```

**验证方式**
`tsc --noEmit` 通过；手动测试：选中节点 A → 直接点击画布节点 B（不关闭 DetailPanel）→ 触发保存 → 验证文件内容写入 B 的正确路径。

### Issue #4: DetailPanel 取消按钮过期闭包

**问题描述**
DetailPanel 编辑状态下点击"取消"按钮，会 reload 原始内容（`storage.readMarkdown(path)`）。`path` 变量使用 `selectedNode?.data.path`，同样受过期闭包影响，导致 reload 到错误的文件。

**原因分析**
同 Issue #3，`selectedNode` 在渲染时捕获，取消按钮处理函数中的路径变量陈旧。

**解决方案**
```tsx
const node = graph.nodesMapRef.current.get(selectedNodeId)
const path = node?.data.path ?? selectedNodeId
```

**验证方式**
`tsc --noEmit` 通过；手动测试流程同 Issue #3，取消后确认重载内容来自正确文件。
