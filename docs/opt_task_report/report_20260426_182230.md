## 八、后续优化建议

1. **测试覆盖率**：项目目前缺少自动化测试。建议为 `graphOperations.ts` 中的节点/边 CRUD 函数和 `useLayout.ts` 的布局计算补充单元测试，提升回归防护能力。

2. **Vite chunk 分割**：ELK.js（438KB gzip）作为单一大 chunk 加载。可考虑通过 `rollupOptions.output.manualChunks` 将 ELK.js 分割为 `vendor-elk`，减小主 bundle 大小并利用浏览器长缓存。

3. **close-guard 模块拆分**：`close-guard.ts` 同时导出 `registerTabSaver`（GraphPage 使用）和 `flushTabs`/`flushAllDirtyTabs`（App 使用），可考虑拆分为 `tab-saver.ts` 和 `close-guard.ts`，减少不必要的共同依赖。

4. **HMR 检查**：确认 `useGraph` 中无 `window.location.reload` 等破坏热更新的调用，确保开发体验流畅。

5. **日志规范统一**：渲染层使用 `logger` / `logAction`，但 preload 中仍存在少量 `console.warn/error`。如需完全统一，需同时修改 Electron 主进程层。
