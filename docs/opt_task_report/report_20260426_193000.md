### 1. 竞态条件保护（3 项）

#### B-D2 — GitPanel requestSeq staleness 保护 ✅

**文件**：`src/components/GitPanel/GitPanel.tsx`

**问题描述**：`useEffect` 中并行请求 `git.status()`、`git.diffFiles()`、`git.remoteGet()` 时缺少 requestSeq 序列号检查。若用户在数据返回前切换知识库分支，异步回调可能基于过期状态更新 UI。

**解决方案**：
1. 添加 `requestSeqRef = useRef(0)` 声明
2. 在 effect 开头执行 `const requestSeq = ++requestSeqRef.current`
3. 在 Promise.all 解析后和 catch 块前均添加 `if (requestSeqRef.current !== requestSeq) return` 守卫

```typescript
const requestSeqRef = useRef(0)
useEffect(() => {
  if (!showGitPanel || !currentKBPath) return
  const requestSeq = ++requestSeqRef.current
  const loadStatus = async () => {
    try {
      const [statusResult, diffResult, remoteResult] = await Promise.all([...])
      if (requestSeqRef.current !== requestSeq) return  // ← staleness guard
      // setStatus, setRemoteUrl, setDiffFiles, setDiffStat
    } catch (e) {
      if (requestSeqRef.current !== requestSeq) return
      logger.catch('GitPanel', 'load git status', e)
    }
  }
  loadStatus()
}, [showGitPanel, currentKBPath, git])
```

**状态**：✅ 已修复

#### B-D3 — DetailPanel handleRenameConfirm stale closure ✅

**文件**：`src/components/DetailPanel/DetailPanel.tsx`

**问题描述**：`handleRenameConfirm` 回调依赖渲染时闭包捕获的 `selectedNode?.data.label`。若用户在确认重命名期间点击其他节点，旧引用可能导致日志记录错误的节点名。

**解决方案**：通过 `graph.nodesMapRef.current.get(selectedNodeId)` 获取最新节点数据，替代渲染时捕获的闭包值。

```typescript
const handleRenameConfirm = () => {
  if (!renameMode || !newName.trim() || !selectedNodeId) {
    setRenameMode(false)
    return
  }
  const node = graph.nodesMapRef.current.get(selectedNodeId)  // ← fresh data
  const oldLabel = node?.data.label ?? selectedNode?.data.label
  const path = node?.data.path ?? selectedNode?.data.path
  logAction('节点:重命名', 'DetailPanel', {
    nodeId: selectedNodeId,
    oldName: oldLabel,   // ← stale-closure-safe
    newName: newName.trim(),
    path,               // ← stale-closure-safe
  })
  graph.renameNode(selectedNodeId, newName.trim())
  setRenameMode(false)
}
```

**状态**：✅ 已修复

#### B2 — StyleSection requestSeq staleness 保护 ✅

**文件**：`src/components/DetailPanel/StyleSection.tsx`

**问题描述**：`useEffect` 中读取 `storage.readConfig()` 后直接调用 `replaceDefaultEdgeStyle`，缺少 requestSeq 检查。当用户在样式编辑过程中切换节点时，可能出现竞态条件。

**解决方案**：与 GitPanel 相同的 requestSeq 模式。

```typescript
const configRequestSeqRef = useRef(0)
useEffect(() => {
  const requestSeq = ++configRequestSeqRef.current
  storage.readConfig().then((config) => {
    if (configRequestSeqRef.current !== requestSeq) return  // ← staleness guard
    if (config.defaultEdgeStyle) {
      replaceDefaultEdgeStyle({...})
    }
  })
}, [storage, replaceDefaultEdgeStyle])
```

**状态**：✅ 已修复

### 2. 工程化优化（2 项）

#### E4 — logger.ts 移除内部 console 调用 ✅

**文件**：`src/core/logger.ts`

**问题描述**：`debug()` 和 `info()` 方法内部直接调用 `console.debug()` 和 `console.info()`，与 logger 作为 `console` 替代品的定位冲突，可能产生双重输出（console 一份 + logWrite IPC 一份）。

**解决方案**：移除 `debug()` 和 `info()` 中的 `console.debug()` / `console.info()` 调用，改为仅通过 `logWrite()` IPC 写入文件日志（与 `warn()` / `error()` 保持一致）。

```typescript
// 修复前
debug(module: string, ...args: unknown[]) {
  console.debug(..._format(module, 'debug'), ...args)  // ← 移除
  logWrite({...})
}

// 修复后
debug(module: string, ...args: unknown[]) {
  if (_shouldLog('debug')) {
    logWrite({
      level: _toServiceLevel('debug'),
      module,
      message: _extractMessage(module, 'debug', ...args),
    }).catch((e) => { _warn('logWrite failed:', e) })
  }
}
```

**状态**：✅ 已修复

### 3. 功能增强（1 项）

#### U2 — ContextMenu 键盘导航 ✅

**文件**：`src/components/ContextMenu/ContextMenu.tsx` + `ContextMenu.module.css`

**问题描述**：右键菜单不支持键盘导航（↑↓ Enter），对键盘用户不友好，与现代无障碍标准不符。

**解决方案**：
1. 添加 `focusedIndex` state 追踪当前聚焦项
2. 添加 `itemButtonRefs` ref 数组追踪各按钮 DOM 元素
3. 使用 `navigableItems = useMemo(...)` 过滤出非分隔线项，使 `focusedIndex` 正确映射
4. 添加 type guard `(item): item is MenuItem => !('separator' in item)` 确保类型安全
5. 合并键盘事件处理到单个 useEffect（Escape 关闭、↑↓ 导航、Enter 激活）
6. 按钮添加 `.focused` CSS 类实现视觉聚焦反馈

```typescript
// Keyboard navigation effect
useEffect(() => {
  const handler = (e: KeyboardEvent) => {
    if (e.key === 'Escape') { e.preventDefault(); onClose() }
    else if (e.key === 'ArrowDown') {
      e.preventDefault()
      setFocusedIndex((prev) => prev < navigableItems.length - 1 ? prev + 1 : 0)
    }
    else if (e.key === 'ArrowUp') {
      e.preventDefault()
      setFocusedIndex((prev) => prev > 0 ? prev - 1 : navigableItems.length - 1)
    }
    else if (e.key === 'Enter' && focusedIndex >= 0 && focusedIndex < navigableItems.length) {
      e.preventDefault()
      const item = navigableItems[focusedIndex]!
      if (item && !item.disabled) item.action()
    }
  }
  document.addEventListener('keydown', handler)
  return () => document.removeEventListener('keydown', handler)
}, [onClose, navigableItems, focusedIndex])
```

**CSS 聚焦样式**：
```css

  background: #e0e8f5;
  outline: 2px solid #1a3a5c;
}
```
