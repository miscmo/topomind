## 本轮新增发现

### 🔍 理论观察（暂不修复）

1. **confirmStore / promptStore Promise settle** — 未被 consume 的 Promise 不会被 reject
   - 组件 unmount 时若 Promise 仍 pending，不会有任何副作用
   - promptStore 的 resolve 来自用户操作（弹窗关闭），confirmStore 同理
   - **建议**：如有需要，在 store 中添加 `cancelPending()` 方法 reject 所有 pending promises
   - 当前不修复：实际场景中组件 unmount 时弹窗已关闭，不构成实际问题

2. **tabStore.ts（449 行）** — 文件规模接近 500 行阈值
   - 当前内聚性良好（所有 Tab 管理逻辑）
   - **建议**：若继续增长，可拆分为 `tabStore.actions.ts` + `tabStore.types.ts`
   - 当前不拆分：过度工程化风险

代码库在本次深度审查范围内达到高水准基线，无任何功能性 bug、安全漏洞或代码异味需要修复。
