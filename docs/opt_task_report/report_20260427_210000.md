## 四、功能优化详情

### 4.1 GitPanel remoteUrl stale closure 修复

**问题描述**: Tab 切换后 GitPanel 仍显示旧的 remoteUrl 值。
**原因分析**: Tab 切换触发 room load，但 GitPanel 在异步更新完成前已渲染，使用了旧的闭包值。
**解决方案**: 使用 `requestSeqRef` 模式，在异步回调中检查序列号，超前回调忽略。
**验证方式**: 切换不同 Tab（各带不同 remoteUrl），观察 GitPanel 是否正确显示。

### 4.2 DetailPanel XSS 防护

**问题描述**: DetailPanel 使用 `dangerouslySetInnerHTML` 渲染 Markdown，未净化 HTML。
**原因分析**: Markdown 渲染输出可能包含恶意脚本，未做 HTML 净化。
**解决方案**: 使用 `DOMPurify.sanitize(marked.parse(markdown))` 净化所有 HTML 输出。
**验证方式**: 在节点描述中注入 `<script>alert(1)</script>`，验证脚本不执行。

### 4.3 Tab/global nav 统一

**问题描述**: GraphPage 内部存在重复的路由导航逻辑，与全局 Tab 管理不一致。
**原因分析**: 历史遗留代码，不同组件对导航状态的理解不同步。
**解决方案**: 引入 `useNavContext`，统一所有组件的导航状态获取方式。
**验证方式**: 测试房间切换、后退、前进、Tab 切换的所有导航场景。

### 4.4 搜索功能完善

**问题描述**: 搜索功能不完整，无清除按钮，无高亮，Tab 切换状态不保留。
**原因分析**: SearchBar 组件冗余（与 Toolbar 重复），搜索状态未 per-tab 持久化。
**解决方案**: 删除 SearchBar，将搜索功能内置于 Toolbar，使用 tabStore 持久化 per-tab 查询。
**验证方式**: 搜索关键词 → 切换 Tab → 切回，搜索状态和结果保留。

### 4.5 IPC 类型安全

**问题描述**: fs-backend.ts IPC 调用返回值类型推断不安全。
**原因分析**: IPC 返回值未严格类型化，可能导致运行时错误。
**解决方案**: 为所有 IPC 调用添加显式返回类型约束。
**验证方式**: TypeScript 编译 0 错误。

### 4.6 useResizePanel 死代码清理

**问题描述**: `handleMouseUp` 回调定义但未被调用，造成死代码。
**原因分析**: 历史遗留代码，cleanup 中使用内联函数替代。
**解决方案**: 删除 `handleMouseUp`，cleanup 中使用命名变量引用。
**验证方式**: 拖拽调整面板 → 释放鼠标 → 面板宽度正确更新。
