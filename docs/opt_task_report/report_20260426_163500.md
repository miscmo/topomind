## 七、工程化问题详情

### 7.1 logger.ts 内部 console 调用

`src/core/logger.ts` 第 57 和 68 行使用了 `console.debug` 和 `console.info`。由于 logger 模块本身就是 console.* 的替代品，这种内部调用会导致双重输出（如果 logger 配置为同时输出到 console 时）。

**建议**：将内部调用改为 logger 自身的 debug/info 方法，或直接使用底层 console.* 内部方法（不经过 _format）。

### 7.2 React Flow 类型断言

`GraphCanvas.tsx:60` 使用 `as KnowledgeNode` 强制断言，暗示 `ReactFlow.Node` 类型与项目 `KnowledgeNode` 类型之间存在类型差异。应在 `src/types/graph.ts` 中确保 `KnowledgeNode` 扩展了 `Node` 的基础类型，或在 `GraphCanvas` 中使用泛型参数 `<KnowledgeNode>` 替代 `Node`。

### 7.3 useNodeActions handleEdgeStyle 类型安全

`handleEdgeStyle` 中对 `JSON.parse(raw)` 的结果直接断言类型，存在运行时错误风险（非法 JSON 已被 catch，但合法 JSON 但结构不对的情况未被处理）。建议使用 Zod schema 验证解析后的对象结构。
