## 三、核心 Bug 修复详情

### B-D1 (CRITICAL) — useResizePanel 面板拖拽完全失效 ✅ 已修复

**文件**：`src/hooks/useResizePanel.ts`

**问题根因**：React 的 `useCallback` Hook 在创建时捕获的是**依赖项当前值的快照**，而非持续更新的引用。

原代码使用 `useState` 存储 `isResizing`：

```typescript
// 原代码（BUG）
const [isResizing, setIsResizing] = useState(false)

const handleMouseMove = useCallback((e: MouseEvent) => {
  if (!isResizing) return  // ← useCallback 创建时捕获 isResizing=false 快照，永远返回！
  // ...拖拽逻辑永远不会执行
}, [isResizing])  // ← isResizing 是 state 值，依赖变化时回调重建
```

`handleMouseDown` 调用 `setIsResizing(true)` 更新了 state，但 `handleMouseMove` 回调在**创建时**捕获的快照是 `false`，即使 deps 变化重建了回调，`mousemove` 事件触发时用的仍是旧的回调引用——**面板拖拽功能完全失效**。

**修复方案**：使用 `useRef` 替代 `useState`：

```typescript
// 修复后
const isResizingRef = useRef(false)

const handleMouseDown = useCallback((e: React.MouseEvent<HTMLDivElement>) => {
  e.preventDefault()
  e.stopPropagation()
  dragStartXRef.current = e.clientX
  dragStartWidthRef.current = initialWidth
  isResizingRef.current = true  // 直接修改 .current，不触发重渲染
}, [initialWidth])

const handleMouseMove = useCallback((e: MouseEvent) => {
  if (!isResizingRef.current) return  // ← 读取 .current，每次都是最新值
  const delta = e.clientX - dragStartXRef.current
  const newWidth = Math.max(minWidth, Math.min(maxWidth, dragStartWidthRef.current - delta))
  onWidthChange(newWidth)
}, [minWidth, maxWidth, onWidthChange])

const handleMouseUp = useCallback(() => {
  isResizingRef.current = false  // ← 直接修改 .current
}, [])
```

`useRef` 的 `.current` 属性在组件整个生命周期中保持同一引用，`handleMouseMove` 每次访问 `isResizingRef.current` 都能拿到最新值，彻底消除 stale closure。

**副作用评估**：`useResizePanel` 返回 `isResizing: isResizingRef.current`，在 `GraphPage.tsx:69` 用于 CSS 类名切换（`className={styles.resizing}`）。由于拖拽操作本身不需要响应式重渲染（鼠标 move 时直接调用 `onWidthChange`），这个返回值仅在首次渲染时读取，之后 CSS 状态通过 body 级别的 `userSelect` / `cursor` 样式体现，不影响功能。

**验证方式**：
1. 打开 TopoMind，进入任意知识库图谱页面
2. 找到右侧面板的 resize 拖拽手柄（分隔线）
3. 按住鼠标左键拖拽，面板宽度应实时变化
4. 拖拽过程中鼠标变为 `col-resize` 光标
5. 释放鼠标，拖拽状态结束
