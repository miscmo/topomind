## 一、严重缺陷修复（CRITICAL）

### 1. `createChildNode` 双写竞态 Bug — 已修复

**问题文件**: `src/hooks/useGraph/graphOperations.ts`
**严重程度**: CRITICAL（数据一致性与竞态条件）

`createChildNode` 在创建新子节点后，对父目录 `_graph.json` 写入了两次，且顺序导致子节点可能不被 `loadRoom` 发现：

```typescript
// BUG（修复前）：两次写入，后一次覆盖前一次
await FSB.writeGraphMeta(parentPath, newMeta)  // 第一次
scheduleSave(dirPath)  // 300ms 后写第二次 — 此时 loadRoom 可能已完成
await loadRoom(reloadPath)
```

**修复方案**: 移除冗余 `scheduleSave`，保留一次直接的 `FSB.writeGraphMeta` 写入并同步更新房间元数据，`loadRoom` 从磁盘重建状态。

### 2. 缺少节点删除后孤立边清理 — 已修复

**问题文件**: `src/hooks/useGraph/graphOperations.ts`
**严重程度**: HIGH（UI 状态不一致）

`applyNodeRemoveChanges` 删除节点后未清理连接在该节点上的边，导致孤立边残留在状态中。

**修复方案**: 删除节点时同时过滤目标边：
```typescript
const removedSet = new Set(changeIds)
const edges = prev.edges.filter((e) =>
  !removedSet.has(e.source) && !removedSet.has(e.target)
)
```
