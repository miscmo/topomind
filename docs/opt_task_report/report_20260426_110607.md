### 1. K1 — macOS Cmd+Backspace 双触发防护 ✅

**文件**：`src/hooks/useKeyboard.ts`

**问题描述**：macOS 上 Cmd+Backspace 组合键会触发两个独立的 Keydown 事件（Meta 键 + Backspace 键）。原代码中 `Delete`/`Backspace` 分支无 Meta 键过滤，可能在单次按键操作中执行两次删除回调。

**解决方案**：在处理函数体最顶部添加 `if (e.metaKey) return` 守卫，忽略 Meta 键对应的独立事件。

```typescript
    if (e.metaKey) return

    const tag = (e.target as HTMLElement).tagName
    if (tag === 'INPUT' || tag === 'TEXTAREA' || (e.target as HTMLElement).isContentEditable) {
      // ...
    }
    // ...
  }
}, ...)
```

### 3. K4 — setActiveTab 无存在性校验 ✅

**文件**：`src/stores/tabStore.ts`

**问题描述**：`setActiveTab(tabId)` 接受任意字符串作为活跃 Tab ID，不做任何存在性校验。当传入不存在的 tabId 时，`activeTabId` 会被设为悬空值，可能导致 UI 行为异常。

**解决方案**：添加存在性检查，只在 Tab 存在时才更新 `activeTabId`。

```typescript
setActiveTab: (tabId: string) => {
  const exists = get().tabs.some((t) => t.id === tabId)
  if (!exists) return
  set({ activeTabId: tabId })
},
```

**状态**：✅ 已修复

### 4. K5 — GitCache._listeners 重复注册内存泄漏 ✅

**文件**：`src/core/git-backend.ts`

**问题描述**：`_listeners` 使用 `Array<...>` 存储，`indexOf` + `splice` 仅移除首次匹配项。若同一回调函数被多次注册，`indexOf` 返回第一个索引，`splice` 仅删除一个元素，剩余副本永久残留于数组中，造成内存泄漏。

**解决方案**：将 `_listeners` 从 `Array` 改为 `Set`，利用 Set 的语义正确性自动去重。

```typescript
// 修复前
const _listeners: Array<(kbPath: string, status: GitCacheStatus | null) => void> = []
onStatusChange(fn) {
  _listeners.push(fn)
  return () => {
    const idx = _listeners.indexOf(fn)  // 只移除首次匹配 ← K5
    if (idx !== -1) _listeners.splice(idx, 1)
  }
}

// 修复后
const _listeners: Set<(kbPath: string, status: GitCacheStatus | null) => void> = new Set()
onStatusChange(fn) {
  _listeners.add(fn)
  return () => {
    _listeners.delete(fn)  // 自动去重，无副本残留
  }
}
```

**状态**：✅ 已修复

### 5. K6 — storage.ts cachedConfig 无 TTL 导致外部修改不感知 ✅

**文件**：`src/core/storage.ts`

**问题描述**：`readConfig()` 无条件每次调用都从 FSB 重新读取文件。虽然无本线程 stale 问题，但缺少 TTL 机制意味着无法高效合并多次调用，且无法区分"外部已修改"与"内存缓存"。

**解决方案**：添加 `cachedConfigTimestamp` 记录缓存时间戳，TTL 为 30000ms，TTL 内直接返回缓存。

```typescript
let cachedConfig: WorkDirConfig = normalizeConfig({})
let cachedConfigTimestamp = 0
const CONFIG_CACHE_TTL = 30000

async readConfig(): Promise<WorkDirConfig> {
  const now = Date.now()
  if (cachedConfigTimestamp && now - cachedConfigTimestamp < CONFIG_CACHE_TTL) {
    return cachedConfig  // TTL 内直接返回缓存
  }
  try {
    cachedConfig = normalizeConfig(await FSB.readAppConfig())
    cachedConfigTimestamp = now
    return cachedConfig
  } catch {
    cachedConfig = normalizeConfig({})
    cachedConfigTimestamp = now
    return cachedConfig
  }
}
```

**状态**：✅ 已修复

### 8. K16 — useKeyboard 回调 Stale Closure 防护 ✅

**文件**：`src/components/GraphPage.tsx`

**问题描述**：`useKeyboard` 的 `onDelete`/`onAddChild` 回调捕获渲染时闭包中的 `nav.selectedNodeId`。若用户在其他地方触发了节点选择变化，这些回调中引用的值可能已过时。

**解决方案**：通过 `useNavContext` 获取 `getNavState`，在回调执行时读取最新状态，而非渲染时捕获。

```typescript
// 修复前
useKeyboard({
  tabId,
  onDelete: () => {
    if (!nav.selectedNodeId) return           // ← K16: 渲染时闭包值
    deleteSelectedNode(nav.selectedNodeId)  // 可能已过时
  },
})

// 修复后
const { getNavState } = useNavContext({ tabId })
useKeyboard({
  tabId,
  onDelete: () => {
    const { selectedNodeId } = getNavState()  // ← K16: 回调执行时最新值
    if (!selectedNodeId) return
    deleteSelectedNode(selectedNodeId)
  },
})
```

**状态**：✅ 已修复

### 9. K17 — HomePage onMouseEnter 日志无防抖 ✅

**文件**：`src/components/HomePage.tsx`

**问题描述**：`onMouseEnter` 绑定无防抖逻辑，鼠标在卡片区域内移动时每像素触发一次 `logAction`。大量不必要的日志写入影响性能，且污染日志数据。

**解决方案**：使用 `Map<string, ReturnType<typeof setTimeout>>` 对每个知识库 path 独立防抖（200ms），避免重复日志。

```typescript
const enterLogTimers = useRef<Map<string, ReturnType<typeof setTimeout>>>(new Map())

onMouseEnter={() => {
  const timers = enterLogTimers.current
  if (timers.has(kb.path)) return
  const timer = setTimeout(() => {
    timers.delete(kb.path)
    logAction('HomePage:悬停知识库卡片', 'HomePage', { kbPath: kb.path, kbName: kb.name })
  }, 200)
  timers.set(kb.path, timer)
}}
```

**状态**：✅ 已修复

### 10. K20 — 直接调用 useAppStore.getState() 替换为 setState Action ✅

**文件**：`src/components/GraphPage.tsx` + `src/hooks/useGraph.ts`

**问题描述**：两处 `onEdgeContextMenu` 中直接调用 `useAppStore.getState().setSelectedEdgeId(edgeId)` 和 `useAppStore.getState().setRightPanelTab('style')`，绕过 Zustand 的 selector 订阅机制，增加不必要的性能开销，且不符合 Zustand v5 最佳实践。

```typescript

onEdgeContextMenu={(edgeId) => {

  useAppStore.getState().setSelectedEdgeId(edgeId)  // ← K20: 绕过订阅

// GraphPage.tsx 修复后
const setSelectedEdgeId = useAppStore((s) => s.setSelectedEdgeId)
onEdgeContextMenu={(edgeId) => {
  if (rightPanelTab !== 'style') setRightPanelTab('style')
  setSelectedEdgeId(edgeId)
}}

// useGraph.ts 修复前
setSelectedEdgeId(edge.id)
useAppStore.getState().setRightPanelTab('style')  // ← K20

// useGraph.ts 修复后
const setRightPanelTab = useAppStore((s) => s.setRightPanelTab)
```
