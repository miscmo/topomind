# Electron \+ React \+ TypeScript 项目注释规范

# 1\. 文档说明

本文档为项目统一注释规范，适用于 Electron 主进程、渲染进程（React）、preload 脚本、工具函数等所有代码文件，目的是统一代码注释风格、提升可读性、降低维护成本，供开发人员及 AI 工具（如 Claude Code）编写注释时遵循。

核心原则：简洁、精准、有用，不写废话；重点标注进程差异、IPC 通信、跨环境逻辑，确保注释与代码同步，AI 生成时需严格匹配代码功能，不虚构逻辑。

# 2\. 通用注释规则（所有文件适用）

## 2\.1 注释风格

- 多行注释：用于文件顶部、函数/类说明，使用 `/\*\* \.\.\. \*/`（JSDoc 规范），换行对齐，关键信息用 @ 标签标注。

- 单行注释：用于代码逻辑说明、注意事项，使用 `// 注释内容`，与代码保持 1 个空格距离，避免行尾注释（除非逻辑极简单）。

- 块级注释：用于 IPC 通信、复杂逻辑块，使用 `// ============================================== \.\.\. // ==============================================` 包裹，突出重点。

## 2\.2 编码要求

- 注释语言：统一使用中文，语法规范、无错别字，避免口语化。

- 敏感信息：禁止在注释中填写密码、密钥、服务器地址等敏感信息。

- 同步更新：代码修改时，对应注释必须同步修改，避免注释与代码不一致。

- AI 生成要求： Claude Code 生成注释时，需结合代码功能，不冗余、不遗漏关键逻辑，严格遵循本文档的标签和格式要求。

# 3\. 分文件/模块注释规范

## 3\.1 文件顶部注释（必写）

所有 \.ts、\.tsx、\.js 文件顶部必须添加注释，标注文件用途、所属进程、作者（可选）、日期（可选），格式统一如下：

```typescript
/**
 * @file 文件名（简要说明文件功能）
 * @description 文件详细用途、核心功能，若为 Electron 相关，需标注所属进程
 * @author 作者名（可选）
 * @date 日期（格式：YYYY-MM-DD，可选）
 * @memberof 所属模块/进程（如：main-process、renderer-process、preload、utils）
 */
```

示例（主进程文件）：

```typescript
/**
 * @file main/window.ts 窗口管理
 * @description 负责 Electron 主窗口的创建、销毁、大小调整、多窗口管理，以及窗口事件监听
 * @author 开发者名称
 * @date 2025-04-23
 * @memberof main-process
 */
```

示例（React 渲染进程文件）：

```typescript
/**
 * @file renderer/components/Home.tsx 首页组件
 * @description 项目首页，负责展示文件列表、调用 Electron 原生 API 打开文件，与主进程通过 IPC 通信
 * @author 开发者名称
 * @date 2025-04-23
 * @memberof renderer-process
 */
```

## 3\.2 Electron 主进程注释规范

主进程核心是窗口管理、IPC 监听、原生 API 调用，注释需重点标注窗口配置、IPC 通信细节、安全配置。

### 3\.2\.1 窗口创建注释

```typescript
/**
 * 创建 Electron 主窗口
 * @description 初始化主窗口配置，区分开发/生产环境加载路径，配置安全参数
 * @returns {BrowserWindow} 主窗口实例
 */
export function createMainWindow() {
  const win = new BrowserWindow({
    width: 1200,
    height: 800,
    webPreferences: {
      nodeIntegration: false, // 关闭 Node 集成，提升安全性（必选）
      contextIsolation: true, // 开启上下文隔离，防止渲染进程篡改主进程（必选）
      preload: path.join(__dirname, "../preload/index.ts"), // 预加载脚本，用于暴露 API
      webSecurity: true, // 开启 Web 安全，禁止跨域（生产环境必选）
    },
  });

  // 开发环境加载 React 本地服务，生产环境加载打包后的 index.html
  if (process.env.NODE_ENV === "development") {
    win.loadURL("http://localhost:3000");
    win.webContents.openDevTools(); // 开发环境自动打开调试工具
  } else {
    win.loadFile(path.join(__dirname, "../../dist/index.html"));
  }

  // 监听窗口关闭事件，释放资源
  win.on("close", () => {
    win.destroy();
  });

  return win;
}
```

### 3\.2\.2 IPC 通信注释（主进程）

所有 ipcMain\.handle / ipcMain\.on 必须添加块级注释，明确通信方向、参数、返回值，格式如下：

```typescript
// ==============================================
// IPC 通信：渲染进程 → 主进程（请求-响应模式）
// 监听事件名：open-file-dialog
// 功能：打开系统文件选择对话框，允许用户选择单个文件
// 参数：无
// 返回值：{ string[] | undefined } 文件路径数组（取消选择则返回 undefined）
// 关联渲染进程：renderer/components/FileOpener.tsx
// ==============================================
ipcMain.handle("open-file-dialog", async () => {
  const { filePaths, canceled } = await dialog.showOpenDialog({
    properties: ["openFile"], // 仅允许选择单个文件
    filters: [{ name: "文本文件", extensions: ["txt", "md"] }], // 文件类型过滤
  });
  return canceled ? undefined : filePaths;
});
```

## 3\.3 Preload 脚本注释规范

Preload 脚本用于安全暴露主进程 API 给渲染进程，注释需标注暴露的 API 用途、调用方式，禁止暴露危险 API。

```typescript
/**
 * @file preload/index.ts 预加载脚本
 * @description 安全地向渲染进程暴露主进程 API，隔离主进程与渲染进程上下文
 * @memberof preload
 */
import { contextBridge, ipcRenderer } from "electron";

// 向渲染进程暴露 API，挂载到 window.electronAPI 上
contextBridge.exposeInMainWorld("electronAPI", {
  // 打开文件对话框（对应主进程 open-file-dialog 事件）
  openFileDialog: () => ipcRenderer.invoke("open-file-dialog"),
  // 保存文件（对应主进程 save-file 事件）
  saveFile: (content: string, filePath?: string) => 
    ipcRenderer.invoke("save-file", content, filePath),
});
```

## 3\.4 React 渲染进程注释规范

渲染进程主要是 React 组件，注释需标注组件用途、props、状态、IPC 调用逻辑，遵循 React 组件注释规范。

### 3\.4\.1 组件注释

```typescript
/**

 * @description 提供文件选择按钮，调用 Electron 原生文件对话框，获取文件路径并展示

 * @props {(path: string[]) => void} onFileSelect - 文件选择后的回调函数
 * @returns {React.ReactElement} 文件选择组件
 */
interface FileOpenerProps {
  defaultPath?: string;
  onFileSelect: (path: string[]) => void;

export default function FileOpener({ defaultPath, onFileSelect }: FileOpenerProps) {
  // 调用 Electron 暴露的 API，打开文件对话框（渲染进程 → 主进程）
  const handleOpenFile = async () => {
    const filePaths = await window.electronAPI.openFileDialog();
    if (filePaths && filePaths.length > 0) {
      onFileSelect(filePaths);
    }

      {defaultPath && 当前文件：{defaultPath}}
```

### 3\.4\.2 Hooks 注释

```typescript
/**
 * 自定义 Hook：监听窗口大小变化
 * @description 监听 Electron 主窗口大小变化，返回当前窗口宽高
 * @returns {[number, number]} [width, height] - 窗口宽高
 */
export function useWindowSize() {
  const [windowSize, setWindowSize] = useState([1200, 800]);

  // 监听窗口大小变化事件（渲染进程监听主进程事件）
  useEffect(() => {
    const handleSizeChange = (event: Electron.IpcRendererEvent, width: number, height: number) => {
      setWindowSize([width, height]);
    };

    ipcRenderer.on("window-size-change", handleSizeChange);
    // 组件卸载时移除监听，避免内存泄漏
    return () => ipcRenderer.removeListener("window-size-change", handleSizeChange);
  }, []);

  return windowSize;
}
```

## 3\.5 工具函数注释规范

工具函数需标注用途、参数、返回值、示例，确保可复用性和可读性，AI 生成时需补充示例。

```typescript
/**
 * 格式化文件大小（字节 → KB/MB/GB）
 * @description 将文件字节数转换为人类可读的格式，保留指定小数位数
 * @param {number} bytes - 文件大小（字节）
 * @param {number} [decimals=2] - 保留的小数位数（默认 2 位）
 * @returns {string} 格式化后的文件大小字符串（如：1.23 MB）
 * @example formatFileSize(1024) → "1.00 KB"
 * @example formatFileSize(1048576, 1) → "1.0 MB"
 */
export function formatFileSize(bytes: number, decimals = 2): string {
  if (bytes === 0) return "0 Bytes";
  const k = 1024;
  const dm = decimals < 0 ? 0 : decimals;
  const sizes = ["Bytes", "KB", "MB", "GB"];
  const i = Math.floor(Math.log(bytes) / Math.log(k));
  return `${parseFloat((bytes / Math.pow(k, i)).toFixed(dm))} ${sizes[i]}`;
}
```

# 4\. 必须写注释的场景（强制要求）

- 所有文件顶部（必写）。

- Electron 主窗口创建、配置。

- 所有 IPC 通信（主进程、渲染进程、preload）。

- preload 脚本暴露的所有 API。

- 自定义 Hooks、工具函数。

- 复杂逻辑块（如条件判断、循环、异步操作）。

- 环境判断（开发/生产环境、主进程/渲染进程）。

- 安全相关配置（如 nodeIntegration、contextIsolation）。

# 5\. 禁止注释的场景

- 废话注释：如 `// 定义变量 a`、`// 点击事件` 等显而易见的逻辑。

- 冗余注释：重复代码逻辑，注释比代码更复杂。

- 过时注释：代码修改后，未同步更新的注释。

- 敏感信息注释：密码、密钥、服务器地址等。

# 6\. AI 生成注释补充要求（给 Claude Code）

- 生成注释时，需先识别代码所属进程（主进程/渲染进程/preload），匹配对应规范。

- IPC 通信注释必须明确通信方向、事件名、参数、返回值，不遗漏关联模块。

- 工具函数、Hooks 必须补充 @example，确保可直接参考使用。

- 注释需与代码逻辑严格对应，不虚构功能、不添加不存在的参数/返回值。

- 遵循本文档的格式、标签要求，不随意修改注释结构。
